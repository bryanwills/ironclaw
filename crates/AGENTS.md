# Crate routing map

Start with domain folders before opening individual crates.

- `foundation/`: shared contracts and substrate vocabulary.
- `policy/`: authority, safety, secrets, network, filesystem, approval gates.
- `runtime/`: capability execution, runtimes, dispatcher, host runtime.
- `state/`: events, run state, threads, conversations, memory, outbound/read models.
- `agent/`: Reborn loop, CLI, LLM, engine, skills.
- `product/`: product adapters and user-facing UIs.
- `meta/`: architecture tests and repo boundary enforcement.
- `legacy/`: excluded/non-core crates.
